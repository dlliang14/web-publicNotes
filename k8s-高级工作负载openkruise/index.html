高级工作负载介绍 #

CloneSet #

  • CloneSet 基于 Deployment 实现的控制器，提供更快的滚动升级和更灵活的发布策略
  • 会发现 age 字段不变，镜像变了。说明是原地升级，没有重建 Pod
# InPlaceIfPossible 升级策略示例：优先尝试原地升级，只有在无法原地升级时才重建 Pod
# partition(5 -> 3 -> 0 ) 字段控制未升级的 Pod 数量，先升级一部分观察效果，再逐步升级剩余 Pod
apiVersion: apps.kruise.io/v1alpha1
kind: CloneSet
metadata:
  name: nginx-test
spec:
  replicas: 5
  updateStrategy:
    type: InPlaceIfPossible  # 优先尝试原地升级
    partition: 3             # 灰度：只升级最后 2 个 Pod (5-3=2)，先观察效果
  template:
    metadata:
      labels:
        app: nginx-test
    spec:
      containers:
      - name: nginx
        image: nginx:1.24  # 修改这里触发升级


# 修改 CloneSet，在 podsToDelete 字段加入那个坏 Pod 的名字
kubectl patch cloneset nginx-test --type='json' -p='[{"op": "add", "path": "/spec/scaleStrategy/podsToDelete", "value":["nginx-test-abcde"]}]'

SidecarSet #

  • SidecarSet 基于 DaemonSet 实现的控制器，支持在现有 Pod 中动态注入或更新 Sidecar 容器，无需重启主容器
# 修改镜像版本，SidecarSet 会自动更新所有匹配的 Pod 中的 filebeat 容器，而不影响主容器的运行
apiVersion: apps.kruise.io/v1alpha1
kind: SidecarSet
metadata:
  name: log-collector
spec:
  selector:
    matchLabels:
      app-type: web-service  # 只要带这个标签的 Pod 都会被注入
  updateStrategy:
    type: RollingUpdate
    maxUnavailable: 20%
  containers:
  - name: filebeat
    image: filebeat:8.0.0
    volumeMounts:
    - name: log-dir
      mountPath: /var/log/app
