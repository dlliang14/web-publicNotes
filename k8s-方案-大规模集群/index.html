灰度发布

  1. apply vs rules 100/0
  2. 修改权重 90/10 验证
  3. 验证不通过 回滚到 100/0
  4. 验证通过 0/100
  5. 云厂商 支持全链路灰度 提供流量泳道（Traffic Lane）
apiVersion: networking.istio.io/v1alpha3
kind: VirtualService
metadata:
  name: my-service-route
spec:
  hosts:
  - my-service           # 目标服务的名称
  http:
  - route:
    - destination:
        host: my-service
        subset: v1       # 对应你例子中的 to: v1
      weight: 80
    - destination:
        host: my-service
        subset: v2       # 对应你例子中的 to: v2
      weight: 20

---

# 1. 基础 Service (只管定义入口)
apiVersion: v1
kind: Service
metadata:
  name: my-service
spec:
  selector:
    app: my-service # 这里的标签要能同时覆盖 v1 和 v2 的 Pod
  ports:
  - port: 80
    targetPort: 8080
---
# 2. 目标规则 (负责版本分组)
apiVersion: networking.istio.io/v1alpha3
kind: DestinationRule
metadata:
  name: my-service-dr
spec:
  host: my-service
  subsets:
  - name: v1
    labels:
      version: v1 # 匹配 v1 Deployment 的 Pod 标签
  - name: v2
    labels:
      version: v2 # 匹配 v2 Deployment 的 Pod 标签

manifest 模板管理 #

Dockerfile 镜像 CICD 管理 #

cicd

  1. 开发人员提交代码到代码仓库（如 GitHub、GitLab 等）。
  2. 触发 CI/CD 流水线，自动构建 Docker 镜像。
  3. 构建完成后，自动将镜像推送到容器镜像仓库（如 Docker Hub、Harbor 等）。
  4. 组装 k8s manifest 文件(资源模板、亲和模板、yaml模板)，更新镜像版本号。
  5. 简单的CD：使用更新后的镜像部署应用。
  6. 复杂的CD：灰度发布，需要摘流；蓝绿发布，deployment滚动到一半时暂停。

Matrix 保护控制器 PDB拓展 增强驱逐 #

  1. PID算法 动态计算超卖比 通知钩子函数修改allocatable
  2. PDB(防止高危驱逐) 分数越高，约容易被驱逐

    2. 可驱逐数N +N分
    3. BestEffort +0.2分

科普 cgroup隔离机制 #

隔离优先级：S10(核心服务) > S20(在线应用) > S30(离线应用)

接口文件对进程组影响样例 (设置值)
cpuset.cpus绑定进程可以运行的具体 CPU 核心编号。用于硬隔离，避免核心竞争。0-2,4 (允许使用核心0,1,2和4)
cpu.shares设置多个 cgroup 占用的 相对权重。只有在 CPU 忙碌时才生效。1024 (默认值，若另一组设为2048则其获取两倍时间)
cpu.cfs_quota_us限制进程组在特定周期内可使用的 最大 CPU 时间（微秒）。-1 表示无限制。50000 (配合周期使用，限制最大占有率)
cpu.cfs_period_us设定统计 CPU 使用时间的 调度周期（微秒）。通常默认为 100ms。100000 (即 100ms)
cpu.max (v2)Cgroup v2 中合并了 quota 和 period 的接口。50000 100000 (等同于 v1 的 50% 限制)
cpu.rt_runtime_us限制 实时进程 (Real-time) 在周期内可以运行的时间，防止实时进程榨干系统。800000 (限制实时任务每秒最多用 0.8s)
cpu.rt_period_us实时进程调度的统计周期。1000000 (即 1s)
cpu.stat只读 接口，查看进程组被限制（Throttled）的具体次数和时长统计。nr_throttled 5 (已被限制了 5 次)
cpuset.mems绑定进程可以使用的 内存节点 (NUMA node)，通常与 cpuset.cpus 配合使用。0 (仅使用内存节点 0)
