Python 生产者消费者

python-生产者消费者 #

python-并发编程 | python-ipc

优点：解耦合，平衡生产者的生产力与消费者的处理能力。

  • 生产者负责产生数据，消费者负责取走数据处理
  • 通过队列通信
  • Python 一般输入 None 作为结束信号

方式1：Queue + None 哨兵 #

主进程在所有生产者结束后，手动向队列投入 None，每个消费者收到 None 后退出。

from multiprocessing import Process, Queue
import time, random

def producer(q, name, food):
    for i in range(3):
        res = f"{food}{i}"
        time.sleep(random.randint(1, 3))
        q.put(res)
        print(f"{name} 生产了 {res}")

def consumer(q, name):
    while True:
        res = q.get()
        if res is None:
            break
        time.sleep(random.randint(1, 3))
        print(f'{name} 吃了 {res}')

if __name__ == '__main__':
    q = Queue()
    producers = [
        Process(target=producer, args=(q, "厨师1", "包子")),
        Process(target=producer, args=(q, "厨师2", "烧麦")),
    ]
    consumers = [
        Process(target=consumer, args=(q, "lxx")),
        Process(target=consumer, args=(q, "hxx")),
    ]
    for p in producers + consumers:
        p.start()
    for p in producers:
        p.join()
    # 每个消费者收到一个 None 就退出
    for _ in consumers:
        q.put(None)

方式2：JoinableQueue（推荐） #

消费者设为守护进程，主进程通过 q.join() 等待所有任务处理完再退出，无需手动发 None。

  • q.join() 阻塞，直到队列中所有任务都被 task_done() 标记完成
  • q.task_done() 消费者处理完一条数据后调用，通知队列
from multiprocessing import Process, JoinableQueue
import time, random

def producer(q, name, food):
    for i in range(3):
        res = f"{food}{i}"
        time.sleep(random.randint(1, 3))
        q.put(res)
        print(f"{name} 生产了 {res}")

def consumer(q, name):
    while True:
        res = q.get()
        time.sleep(random.randint(1, 3))
        print(f'{name} 吃了 {res}')
        q.task_done()

if __name__ == '__main__':
    q = JoinableQueue()
    producers = [
        Process(target=producer, args=(q, "厨师1", "包子")),
        Process(target=producer, args=(q, "厨师2", "烧麦")),
    ]
    consumers = [
        Process(target=consumer, args=(q, "lxx")),
        Process(target=consumer, args=(q, "hxx")),
    ]
    for c in consumers:
        c.daemon = True  # 守护进程，主进程结束时自动退出
    for p in producers + consumers:
        p.start()
    for p in producers:
        p.join()
    q.join()  # 等待所有任务被消费完
