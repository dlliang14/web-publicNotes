1.3.1. 元数据要求 #

  • Hugo 对文件元数据要求很高。请确保标题使用双引号括起来，并在文件顶部的 front matter 中定义标题
  • 可选 front matter 配置项
    • type = “docs” 页面按 docs 模板渲染
    • weight：左侧排序权重
    • bookFlatSection：是否扁平化 section
    • bookCollapseSection：该层是否默认折叠子项
    • bookHidden：是否从左侧菜单隐藏
    • bookToC：是否显示右侧目录
    • bookComments：是否启用评论
    • bookSearchExclude：是否排除搜索
    • bookHref：菜单点击跳转外链
# 可以通过 title 或者 一级标题 来设置标题
---
title: "Example Title"
tags:
- example-tag
---

# 左侧不想显示的页面可以设置 bookHidden: true
# 我把非大纲的都设置了 bookHidden: true

# Hugo 通用字段
常见有：title、date、draft、tags、categories、aliases、summary、description、slug、url、lastmod 等。

1.4. hugo 预览 #

windows命令行工具

# Install and link `hugo-obsidian` locally
go install github.com/jackyzha0/hugo-obsidian@latest

cd ./amethyst

# 本地预览
# 需要有make工具，可以参考双链教程安装
make serve

# 会有动态更新的文件，需要加入.gitignore
resources/_gen  # 这是 Hugo 的 处理缓存（如缩减后的 CSS、转换后的 SCSS、裁剪后的图片）
assets/indices/*.json # 这是 hugo-obsidian 生成的索引文件
static/linkmap # 这通常是 Quartz 映射双链的中间产物


# 如果之前已经被 Git 跟踪，还需要把它从索引里移除（保留本地文件）
git rm --cached -r resources/_gen
git rm --cached -r assets/indices/*.json
git rm --cached -r static/linkmap

github actions 自动部署 #

参考文字 hugo 忽略渲染逻辑

  • on.push: 监听 main 分支的 push 事件
  • jobs.deploy: 定义部署流程
    • runs-on: ubuntu-latest 选择运行环境
    • actions/checkout@v4
    • jackyzha0/hugo-obsidian@v2.20
    • peaceiris/actions-hugo@v3
    • build web
    • peaceiris/actions-gh-pages@v3

# 创建token - 务必勾选上图权限

# config.yaml
baseURL: https://dlliang14.github.io/amethyst/

# 忽略渲染 方案1 config.yaml
ignoreFiles: [  
    "/content/.foam/templates/*",  
    "/content/private/*", 
    "<your path here>"
]

# 忽略渲染 方案2 front matter 中设置
# 但是 Interactive Graph 里还是有啊！好丑！
draft: true
# 本地想看看可以设置`--buildDrafts|-D`
hugo server --buildDrafts
# 不含草稿
hugo --minify

# .github/workflows/deploy.yaml
          cname: dlliang14.github.io

# config.yaml 换个新分支来发布！
          publish_branch: deploy  # deploying branch


# 坑点！我是有子仓库的，不加 submodules: recursive 会导致子仓库内容无法拉取，发布后页面是空的
      - name: Checkout
        uses: actions/checkout@v4
        with:
          fetch-depth: 0 # Fetch all history for .GitInfo and .Lastmod
          submodules: recursive
          token: ${{ secrets.PUBLISH_TOKEN }}
