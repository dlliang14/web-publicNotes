线上故障排查-系统资源分析 #

线上故障排查

CPU 与负载 #

  • top：看进程/线程热点
  • vmstat -w 1：看运行队列与阻塞
  • mpstat -P ALL 1：看各核是否倾斜

常用命令怎么用 #

uptime
dmesg | tail
mpstat -P ALL 1
free -m
top
  • uptime：最快速看 load average 1/5/15，适合先判断机器是否整体过载。
  • dmesg | tail：看最近内核日志，常用于确认 OOM、磁盘报错、网卡异常、文件系统错误。
  • mpstat -P ALL 1：-P ALL 表示看所有 CPU 核；后面的 1 表示每 1 秒刷一次，适合看是否单核打满或负载倾斜。
  • free -m：-m 以 MB 展示，适合快速看可用内存、缓存和 swap。
  • top：综合观察入口，适合先锁定异常进程，再决定是否切到线程或磁盘/网络维度。

top 常用交互 #

  • 1：展开或折叠多核 CPU 视图
  • P：按 CPU 排序
  • M：按内存排序
  • c：显示完整命令路径
  • H：切换线程模式
  • <、>、R：切换排序列和排序方向
  • o：自定义过滤器，例如 COMMAND=java

关键字段：

  • load average：负载趋势
  • %us/%sy/%wa/%st：用户态、内核态、IO 等待、虚拟化偷取
  • r/b：CPU 队列与 IO 阻塞

load average 告警参考 #

设逻辑核心数为 $N$：

  • 预警：load_15 > N * 0.7
  • 严重：load_5 > N * 1.0
  • 紧急：load_1 > N * 2.0

这三个窗口分别对应趋势、持续满载和短时雪崩信号。

CPU 与内存字段解释 #

  • %us：用户态 CPU
  • %ni：带 nice 优先级的用户态 CPU
  • %sy：内核态 CPU
  • %wa：等待 IO 的 CPU 时间
  • %st：被虚拟化平台偷走的 CPU 时间
  • %hi/%si：硬件中断和软件中断开销
  • %id：空闲 CPU
  • total/used/free/buff/cache/avail：总量、已用、空闲、缓存、可用内存

磁盘 #

  • iostat -xz 1 关注 %util、r_await/w_await、aqu-sz
  • iotop -P -o 定位具体高 IO 进程

iostat 命令说明 #

iostat -xz 1
  • -x：显示扩展统计字段，例如 await、aqu-sz、%util。
  • -z：隐藏没有活动的设备，避免输出噪音。
  • 1：每秒刷新一次，便于看实时波动。

常看字段：

  • r/s、w/s：每秒读写请求数，可近似理解为 IOPS。
  • rkB/s、wkB/s：每秒读写吞吐量。
  • r_await、w_await：请求平均等待时间，偏高通常说明磁盘响应慢或队列积压。
  • aqu-sz：平均队列长度，持续偏高说明请求排队严重。
  • %util：设备忙碌程度，不等同于“百分百打满”，尤其在 SSD 或 RAID 上要结合 await、队列长度一起看。

iotop 命令说明 #

iotop -P -o
  • -P：按进程显示，不展开到线程。
  • -o：只显示有 IO 活动的进程。

适合在已经确认“磁盘忙”之后，继续定位到底是哪个进程在写日志、刷盘、落快照或做大批量扫描。

补充判断：

  • 第一行通常是历史累计值，分析实时问题时可忽略。
  • SSD 或 RAID 存在并发处理能力，%util 可能超过 100%，不能机械理解为“已经打满”。
  • r/s、w/s 反映 IOPS；aqu-sz 反映平均队列长度；await 反映请求等待时延。

内存 #

  • free -m 看可用内存与 swap
  • slabtop -s c 看内核对象占用

slabtop 命令说明 #

slabtop -s c
  • slabtop：查看内核 slab 分配器中的对象占用。
  • -s c：按缓存对象数量排序。

它适合回答一个问题：buff/cache 很大时，到底是哪类内核对象在吃内存，例如 dentry、inode、network buffer 等。

补充判断：

  • slabtop 适合定位 buffer/cache 大时，到底是哪类内核对象吃掉了内存。
  • swap 一旦持续增长，通常意味着回收压力已经比较明显。

vmstat #

vmstat -w 1

  • -w：宽输出，避免字段太长被截断。

  • 1：每秒输出一次。

  • 第一行是启动以来累计值，通常不用看。

  • r 高说明 CPU 饱和，运行队列积压。

  • b 高说明大量进程在等 IO。

  • si/so 反映 swap 进出速率。

  • bi/bo 反映块设备读写速率。

  • in/cs 反映中断和上下文切换频率。

典型用法：

  • r 明显大于 CPU 核数，同时 %id 很低：更偏 CPU 打满。
  • b 很高，同时 %wa 很高：更偏磁盘或网络 IO 卡顿。
  • si/so 持续不为 0：说明机器已经开始用 swap 顶内存压力。

网络 #

  • sar -n DEV 1 看吞吐
  • sar -n EDEV 1 看丢包与错误
  • iftop -B -nNP 看连接维度流量
  • netstat/ss 看连接状态异常（TIME_WAIT、CLOSE_WAIT、SYN_RECV）

sar 观测重点 #

  • sar -u 1：整体 CPU
  • sar -d -p 1：磁盘性能
  • sar -n DEV 1：网卡流量
  • sar -n EDEV 1：网卡错误和丢包
  • sar -n TCP 1：TCP 连接统计
  • sar -w 1：上下文切换

其中 rxdrop/s、txdrop/s、fifov/s 是排查网卡缓冲区打满时最有价值的字段。

iftop 命令说明 #

iftop -B -nNP
  • -B：流量单位按 Byte 显示，不用 bit。
  • -n：IP 不做 DNS 解析，避免慢和干扰。
  • -N：端口不转服务名，直接显示数字端口。
  • -P：显示端口信息。

适合在“网卡很忙”时继续追到连接维度，定位是哪一批源 IP、目标 IP、端口在占带宽。

netstat / ss 命令说明 #

netstat -antp | awk '{print $6}' | sort | uniq -c | sort -nr
ss -ant | awk '{print $1}' | sort | uniq -c
  • -a：显示所有连接。
  • -n：地址和端口都用数字显示。
  • -t：只看 TCP。
  • -p：显示所属进程。

这两条命令的目的不是看单条连接，而是做“状态分布统计”，快速判断是否存在大量 TIME_WAIT、CLOSE_WAIT、SYN_RECV 等异常状态。

连接状态判断 #

  • SYN_RECV 多：半连接积压，可能是 SYN flood、网络丢包或 accept 跟不上
  • TIME_WAIT 多：通常是主动断开过多，可能是下游不稳或连接复用不足
  • CLOSE_WAIT 多：通常是本服务没有正常关闭连接，更偏应用代码问题

告警建议 #

  • 负载、延迟、丢包应设置分级阈值（预警/严重/紧急）
  • 告警要配排障动作，不要只给数字
