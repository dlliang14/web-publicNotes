k8s-部署-通用注意点 #

1. 机器准备 #

  1. 磁盘分区，最好lvm，文件系统确认支不支持k8s的存储驱动(xfs,ext4)
  2. 内核升级
  3. 禁用 Swap 与 SELinux
  4. 内核参数优化
  5. 关闭防火墙
  6. 开启必备模块
  7. 时间同步
  8. 离线环境，需要准备好离线包，离线镜像仓库或者直接load镜像到本地

2. 内核版本以及优化 #

  • 内核版本不一样，相关参数可能有区别
  • 下面的脚本参考的kubekey,有 awk ' !x[$0]++{print > "'$tmpfile'"}' /etc/sysctl.conf 主要是去重，先提前解释一下

2.1. 禁用 Swap 与 SELinux #

swapoff -a
sed -i /^[^#]*swap*/s/^/\#/g /etc/fstab

# See https://github.com/kubernetes/website/issues/14457
if [ -f /etc/selinux/config ]; then
  sed -ri 's/SELINUX=enforcing/SELINUX=disabled/' /etc/selinux/config
fi
# for ubuntu: sudo apt install selinux-utils
# for centos: yum install selinux-policy
if command -v setenforce &> /dev/null
then
  setenforce 0
  getenforce
fi

2.2. 内核参数优化 #

参数名称常见默认值优化后建议值功能与优化说明
net.core.netdev_max_backlog100065535接口接收数据包的队列长度。增加该值可防止在高流量冲击时因 CPU 处理不及时导致丢包。
net.core.rmem_max~212 KB33554432 (32MB)socket 接收缓冲区的最大字节数，提升大流量下的数据接收吞吐量。
net.core.wmem_max~212 KB33554432 (32MB)socket 发送缓冲区的最大字节数，提升大流量下的数据发送吞吐量。
net.core.somaxconn12832768TCP “全连接队列”（accept 队列）的最大长度。对 Nginx、Redis 等高并发应用至关重要，防止连接溢出。
net.bridge.bridge-nf-call-arptables01开启后使桥接流量经过 arptables 过滤，常用于容器网络（Docker/K8s）的策略控制。
vm.max_map_count65530262144限制一个进程拥有的虚拟内存映射区域数量。Elasticsearch 等数据库强制要求调大此值。
vm.swappiness600控制物理内存剩余多少时使用 Swap。设为 0 表示最大限度使用物理内存，降低磁盘 I/O 延迟。
vm.overcommit_memory01允许内核分配超过物理内存总额的虚拟内存。解决 Redis 等应用在 fork 进程时申请内存失败的问题。
fs.inotify.max_user_instances128524288增加用户可创建的 inotify 实例上限，防止监控服务或大量容器因资源受限而崩溃。
fs.inotify.max_user_watches819210240001单个实例可监控的文件/目录数量。在大文件系统监控或代码构建场景下必须调大。
fs.pipe-max-size1 MB4194304 (4MB)提升进程间通信（Pipe）的单次传输效率，减轻系统调用负担。
fs.aio-max-nr65536262144增加系统级并发异步 I/O (AIO) 请求上限，对高性能数据库读写有显著帮助。
kernel.pid_max3276865535提升系统最大进程 ID 限制。在微服务极其密集或线程极多的环境下，防止无法创建新进程。
kernel.watchdog_thresh105软死锁（Soft Lockup）检查阈值。缩短时间可以更早发现内核线程卡死情况。
kernel.hung_task_timeout_secs1205如果进程处于 D 状态（不可中断睡眠）超过此时间则报错，有助于快速定位 I/O 挂起问题。
echo 'net.core.netdev_max_backlog = 65535' >> /etc/sysctl.conf
echo 'net.core.rmem_max = 33554432' >> /etc/sysctl.conf
echo 'net.core.wmem_max = 33554432' >> /etc/sysctl.conf
echo 'net.core.somaxconn = 32768' >> /etc/sysctl.conf
echo 'net.bridge.bridge-nf-call-arptables = 1' >> /etc/sysctl.conf
echo 'vm.max_map_count = 262144' >> /etc/sysctl.conf
echo 'vm.swappiness = 0' >> /etc/sysctl.conf
echo 'vm.overcommit_memory = 1' >> /etc/sysctl.conf
echo 'fs.inotify.max_user_instances = 524288' >> /etc/sysctl.conf
echo 'fs.inotify.max_user_watches = 10240001' >> /etc/sysctl.conf
echo 'fs.pipe-max-size = 4194304' >> /etc/sysctl.conf
echo 'fs.aio-max-nr = 262144' >> /etc/sysctl.conf
echo 'kernel.pid_max = 65535' >> /etc/sysctl.conf
echo 'kernel.watchdog_thresh = 5' >> /etc/sysctl.conf
echo 'kernel.hung_task_timeout_secs = 5' >> /etc/sysctl.conf

2.3. ipv4 参数优化 #

  • arp
参数名常见默认值优化后建议值功能与优化说明
net.bridge.bridge-nf-call-iptables01允许桥接流量经过 iptables。K8s Service 与安全策略的底层基石。
net.ipv4.ip_forward01开启 IP 转发。允许宿主机网卡与容器虚拟网卡间传递数据包。
net.ipv4.tcp_tw_recycle00关闭快速回收。避免在 NAT 环境下由于时间戳不一致导致连接被拒。
net.ipv4.tcp_tw_reuse00是否重用 TIME-WAIT 连接。K8s 环境建议关闭或保持默认。
net.ipv4.tcp_max_tw_buckets随内存1048576最大 TIME_WAIT 数量。增大此值可承载更高并发的短连接。
net.ipv4.tcp_max_syn_backlog1281048576增大半连接池。防止在高负载或 SYN 攻击下导致连接超时。
net.ipv4.tcp_retries21515TCP 重传次数。控制连接超时的具体时长（15次约为15分钟）。
net.ipv4.neigh.default.gc_thresh1128512ARP 缓存表最小阈值。低于此值内核不进行清理。
net.ipv4.neigh.default.gc_thresh25122048ARP 缓存软限制。超过此值且满 5 秒则进行清理。
net.ipv4.neigh.default.gc_thresh310244096ARP 硬限制。超过此值立即清理，集群规模大时必须调优。
net.ipv4.ip_local_reserved_ports空30000-32767预留端口范围。防止系统随机分配端口时抢占了 K8s NodePort。
net.ipv4.conf.all.rp_filter01反向路径校验。开启可防止源地址欺骗，增强网络安全性。
net.ipv4.conf.all.arp_accept01开启后可根据接收到的 ARP 广播主动更新缓存，减少查询延迟。
net.ipv4.conf.all.arp_ignore01仅对目标 IP 配置在入接口上的 ARP 请求进行响应，防止路由混乱。
echo 'net.bridge.bridge-nf-call-iptables = 1' >> /etc/sysctl.conf
echo 'net.ipv4.tcp_tw_recycle = 0' >> /etc/sysctl.conf
echo 'net.ipv4.tcp_tw_reuse = 0' >> /etc/sysctl.conf
echo 'net.ipv4.ip_forward = 1' >> /etc/sysctl.conf
echo 'net.ipv4.ip_local_reserved_ports = 30000-32767' >> /etc/sysctl.conf
echo 'net.ipv4.tcp_max_syn_backlog = 1048576' >> /etc/sysctl.conf
echo 'net.ipv4.neigh.default.gc_thresh1 = 512' >> /etc/sysctl.conf
echo 'net.ipv4.neigh.default.gc_thresh2 = 2048' >> /etc/sysctl.conf
echo 'net.ipv4.neigh.default.gc_thresh3 = 4096' >> /etc/sysctl.conf
echo 'net.ipv4.tcp_retries2 = 15' >> /etc/sysctl.conf
echo 'net.ipv4.tcp_max_tw_buckets = 1048576' >> /etc/sysctl.conf
echo 'net.ipv4.tcp_max_orphans = 65535' >> /etc/sysctl.conf
echo 'net.ipv4.udp_rmem_min = 131072' >> /etc/sysctl.conf
echo 'net.ipv4.udp_wmem_min = 131072' >> /etc/sysctl.conf
echo 'net.ipv4.conf.all.rp_filter = 1' >> /etc/sysctl.conf
echo 'net.ipv4.conf.default.rp_filter = 1' >> /etc/sysctl.conf
echo 'net.ipv4.conf.all.arp_accept = 1' >> /etc/sysctl.conf
echo 'net.ipv4.conf.default.arp_accept = 1' >> /etc/sysctl.conf
echo 'net.ipv4.conf.all.arp_ignore = 1' >> /etc/sysctl.conf
echo 'net.ipv4.conf.default.arp_ignore = 1' >> /etc/sysctl.conf

2.4. ipv6 参数优化 #

参数名常见默认值优化建议值功能与优化说明
net.bridge.bridge-nf-call-ip6tables01核心参数。确保 IPv6 桥接流量经过 ip6tables 处理。这对于 IPv6 模式下的 NetworkPolicy 和 Service 负载均衡是必须的。
net.ipv6.conf.all.disable_ipv60 (或 1)0强制开启 IPv6。确保所有网卡（all）都不禁用 IPv6 协议栈，这是双栈集群的基础。
net.ipv6.conf.default.disable_ipv600默认开启 IPv6。确保新创建的虚拟网卡（如容器的 veth）默认加载 IPv6 协议栈。
net.ipv6.conf.lo.disable_ipv600回环口开启 IPv6。确保本地回环接口可以处理 ::1 等 IPv6 流量，防止内部通信失败。
net.ipv6.conf.all.forwarding01IPv6 路由转发。允许宿主机转发目的地不是自己的 IPv6 数据包。没有它，Pod 就无法通过宿主机访问外网。
net.ipv6.conf.default.accept_dad10禁用 DAD (重复地址检测)。在容器频繁启停的环境下，禁用 DAD 可以加快虚拟网卡获取 IPv6 地址的速度，减少 Pod 启动延迟。
net.ipv6.route.max_size409665536增大路由表容量。在大型集群中，IPv6 路由条目可能非常多，增大此值可防止路由表溢出导致的“网络不可达”错误。
net.ipv6.neigh.default.retrans_time_ms10001000邻居发现重传间隔。设置 NS（邻居请求）数据包的重传时间。保持在 1s 可以确保在网络波动时较快地重新发现邻居 MAC 地址。
echo 'net.bridge.bridge-nf-call-ip6tables = 1' >> /etc/sysctl.conf
echo 'net.ipv6.conf.all.disable_ipv6 = 0' >> /etc/sysctl.conf
echo 'net.ipv6.conf.default.disable_ipv6 = 0' >> /etc/sysctl.conf
echo 'net.ipv6.conf.lo.disable_ipv6 = 0' >> /etc/sysctl.conf
echo 'net.ipv6.conf.all.forwarding=1' >> /etc/sysctl.conf
echo 'net.ipv6.conf.default.accept_dad=0' >> /etc/sysctl.conf
echo 'net.ipv6.route.max_size=65536' >> /etc/sysctl.conf
echo 'net.ipv6.neigh.default.retrans_time_ms=1000' >> /etc/sysctl.conf

2.5. ulimit 参数优化 #

  • nofile 单个进程可打开的最大文件句柄数
  • nproc 单个用户可创建的最大进程数
  • memlock 进程可锁定在内存中的最大字节数 不是限制最大内存使用量
echo "* soft nofile 1048576" >> /etc/security/limits.conf
echo "* hard nofile 1048576" >> /etc/security/limits.conf
echo "* soft nproc 65536" >> /etc/security/limits.conf
echo "* hard nproc 65536" >> /etc/security/limits.conf
echo "* soft memlock unlimited" >> /etc/security/limits.conf
echo "* hard memlock unlimited" >> /etc/security/limits.conf

sed -r -i "s@#{0,}?\* soft nofile ?([0-9]{1,})@\* soft nofile 1048576@g" /etc/security/limits.conf
sed -r -i "s@#{0,}?\* hard nofile ?([0-9]{1,})@\* hard nofile 1048576@g" /etc/security/limits.conf
sed -r -i "s@#{0,}?\* soft nproc ?([0-9]{1,})@\* soft nproc 65536@g" /etc/security/limits.conf
sed -r -i "s@#{0,}?\* hard nproc ?([0-9]{1,})@\* hard nproc 65536@g" /etc/security/limits.conf
sed -r -i "s@#{0,}?\* soft memlock ?([0-9]{1,}([TGKM]B){0,1}|unlimited)@\* soft memlock unlimited@g" /etc/security/limits.conf
sed -r -i "s@#{0,}?\* hard memlock ?([0-9]{1,}([TGKM]B){0,1}|unlimited)@\* hard memlock unlimited@g" /etc/security/limits.conf

tmpfile="$$.tmp"
awk ' !x[$0]++{print > "'$tmpfile'"}' /etc/security/limits.conf
mv $tmpfile /etc/security/limits.conf

4. 开启必备模块 #

  • br_netfilter 桥接流量经过 iptables
  • overlay 分层存储驱动（Overlay2）
  • ip_vs IPVS 负载均衡
  • ip_vs_rr 轮询
  • ip_vs_wrr 加权轮询
  • ip_vs_sh 源地址哈希
  • nf_conntrack_ipv4 连接跟踪
  • 不同内核版本模块名称不同
    • ip_conntrack -> nf_conntrack_ipv4 -> nf_conntrack
modinfo br_netfilter > /dev/null 2>&1
if [ $? -eq 0 ]; then
   modprobe br_netfilter
   mkdir -p /etc/modules-load.d
   echo 'br_netfilter' > /etc/modules-load.d/kubekey-br_netfilter.conf
fi

modinfo overlay > /dev/null 2>&1
if [ $? -eq 0 ]; then
   modprobe overlay
   echo 'overlay' >> /etc/modules-load.d/kubekey-br_netfilter.conf
fi

modprobe ip_vs
modprobe ip_vs_rr
modprobe ip_vs_wrr
modprobe ip_vs_sh

cat > /etc/modules-load.d/kube_proxy-ipvs.conf <<EOF
ip_vs
ip_vs_rr
ip_vs_wrr
ip_vs_sh
EOF

modprobe nf_conntrack_ipv4 1>/dev/null 2>/dev/null
if [ $? -eq 0 ]; then
   echo 'nf_conntrack_ipv4' >> /etc/modules-load.d/kube_proxy-ipvs.conf
else
   modprobe nf_conntrack
   echo 'nf_conntrack' >> /etc/modules-load.d/kube_proxy-ipvs.conf
fi
sysctl -p

sync
echo 3 > /proc/sys/vm/drop_caches
