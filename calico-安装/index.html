Calico 安装

calico-安装 #

安装所需条件 - v3.31 #

官网介绍

  • linux内核版本 uname -r 6.8.0-90-generic
  • 系统版本 cat /etc/os-release PRETTY_NAME="Ubuntu 24.04.2 LTS"
  • k8s 版本 kubectl version Server Version: v1.32.4
  • 开启arp_proxy
  • calico 如果要支持ebpf 需要内核版本 >= 5.3 ；k8s 版本 >= 1.21；calico 版本 >= 3.13
# 内核 Linux kernel 5.10 or later
uname -r
# 6.8.0-90-generic

sysctl -w net.ipv4.conf.all.arp_proxy=1
sysctl -w net.ipv4.conf.default.arp_proxy=1

calico 安装 #

官网安装步骤

  1. 安装 crd sa role binding
  2. 安装 custom-resources: 先下载，再修改installation
    1. 查看当前集群的cidr 主要看 cluster-cidr pod网段
    2. 外加自己设置blocksize
# 安装 crd sa role binding
wget https://raw.githubusercontent.com/projectcalico/calico/v3.31.3/manifests/tigera-operator.yaml

# 安装 custom-resources 先下载，再修改installation
wget https://raw.githubusercontent.com/projectcalico/calico/v3.31.3/manifests/custom-resources.yaml
        blockSize: 26
        cidr: 192.168.0.0/16
to
        blockSize: 24
        cidr: 172.16.0.0/16


# 查看当前集群的cidr
# 主要看 cluster-cidr pod网段
kubectl cluster-info dump | grep -m 1 cluster-cidr
                            "--cluster-cidr=172.16.0.0/16",

kubectl cluster-info dump | grep -m 1 service-cluster-ip-range
                            "--service-cluster-ip-range=172.18.0.0/16",

# 或者查看cm kubeadm-config -o yaml
    networking:
      dnsDomain: cluster.local
      podSubnet: 172.16.0.0/16
      serviceSubnet: 172.18.0.0/16

# 外加自己设置blocksize
# 查看当前配置node最多多少pod
# 默认一般110个 由每个节点上的 kubelet 服务控制
kubectl get nodes -o json | jq '.items[] | {name: .metadata.name, pods: .status.capacity.pods}'
{
  "name": "prod-m1",
  "pods": "110"
}
# 110 个pod，2^7=128 >110 设置 blocksize 25 就可以了

# 如果想调大这个限制
vi /var/lib/kubelet/config.yaml
# maxPods: 128
systemctl restart kubelet  # 会有瞬间的event告警

# 安装 calico
kubectl create -f tigera-operator.yaml
kubectl create -f custom-resources.yaml

# 查看安装状态
watch kubectl get tigerastatus

# This section includes base Calico installation configuration.
# For more information, see: https://docs.tigera.io/calico/latest/reference/installation/api#operator.tigera.io/v1.Installation
apiVersion: operator.tigera.io/v1
kind: Installation
metadata:
  name: default
spec:
  # Configures Calico networking.
  calicoNetwork:
    ipPools:
      - name: default-ipv4-ippool
        blockSize: 26
        cidr: 192.168.0.0/16
        encapsulation: VXLANCrossSubnet
        natOutgoing: Enabled
        nodeSelector: all()

  name: tigera-operator
  namespace: tigera-operator

  template:

      labels:
        name: tigera-operator
        k8s-app: tigera-operator
    spec:
      nodeSelector:
        kubernetes.io/os: linux
      tolerations:
        - effect: NoExecute
          operator: Exists
        - effect: NoSchedule
          operator: Exists
      serviceAccountName: tigera-operator
      # Set the termination grace period to match how long the operator will wait for
      # resources to terminate when being uninstalled.
      terminationGracePeriodSeconds: 60
      hostNetwork: true
      # This must be set when hostNetwork is true or else the cluster services won't resolve
      dnsPolicy: ClusterFirstWithHostNet
      containers:
        - name: tigera-operator
          image: quay.io/tigera/operator:v1.40.3
          imagePullPolicy: IfNotPresent
          command:
            - operator
          args:
            # Configure tigera-operator to manage installation of the necessary CRDs.
            - -manage-crds=true
          volumeMounts:
            - name: var-lib-calico
              readOnly: true
              mountPath: /var/lib/calico
          env:

              value: ""
            - name: POD_NAME
              valueFrom:
                fieldRef:
                  fieldPath: metadata.name
            - name: OPERATOR_NAME
              value: "tigera-operator"
            - name: TIGERA_OPERATOR_INIT_IMAGE_VERSION
              value: v1.40.3

            - configMapRef:
                name: kubernetes-services-endpoint
