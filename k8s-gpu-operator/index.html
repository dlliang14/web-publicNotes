2. 组件分层理解 #

2.1. 核心控制层 (The Brain) #

  • gpu-operator-xxx (Deployment)：核心控制器，负责监控 GPU 资源状态并协调其他组件的部署和更新。

2.2. 硬件感知层 (NFD: Node Feature Discovery) #

  • gpu-operator-node-feature-discovery-master-xxx (Deployment)：NFD 的中心节点，负责收集汇总所有 Worker 节点上报的硬件信息
  • gpu-operator-node-feature-discovery-worker-xxx (DaemonSet)：在每个节点上运行，探测物理硬件（PCI 设备、CPU 指令集等）。它会发现节点上显卡，并给节点打上标签（如 feature.node.kubernetes.io/pci-10de.present=true）
  • gpu-operator-node-feature-discovery-gc-xxx (Deployment)：垃圾回收，负责清理过时或无用的节点标签。
  • gpu-feature-discovery-xxx (DaemonSet)：专门针对 NVIDIA GPU 的标签生成器。它会将显卡型号、显存大小等详细信息变成节点标签（例如 nvidia.com/gpu.product=GeForce-RTX-3070）

2.3. 基础架构注入层 (The Heavy Lifters) #

  • nvidia-driver-daemonset-xxx (DaemonSet)：安装驱动。它在容器里编译并向宿主机内核加载 NVIDIA 内核模块。这就是你之前报错“Secure Boot”的地方。
  • nvidia-container-toolkit-daemonset-xxx (DaemonSet)：安装容器运行时。它把 nvidia-container-runtime 塞进宿主机，并修改 containerd 的配置文件，让容器能调用显卡。
  • nvidia-device-plugin-daemonset-xxx (DaemonSet)：GPU 资源暴露。它告诉 K8s 调度器：“这台机器有 1 个可用 GPU”。没有它，你的 Pod 就会卡在 Pending 状态。

2.4. 验证与监控层 (Quality Assurance) #

  • nvidia-operator-validator-xxx (Deployment)：综合自检。它会等待驱动和工具箱装好，然后运行一系列检查，确保整个链条是通的。
  • nvidia-cuda-validator-xxx (Job)：CUDA 测试。它会运行一个小的 CUDA 程序，验证 GPU 是否能正常执行计算任务。
  • nvidia-dcgm-exporter-xxx (DaemonSet)：性能监控。它部署 NVIDIA DCGM（Data Center GPU Manager）Exporter，收集 GPU 的性能指标并暴露给 Prometheus。

3. 安装 #

helm repo add nvidia https://helm.ngc.nvidia.com/nvidia
helm repo update
helm pull --untar nvidia/gpu-operator
helm install gpu-operator . -n gpu-operator --create-namespace

4. 常见故障：Secure Boot #

  • 现象：modprobe: ERROR: could not insert 'nvidia': Key was rejected by service
  • 根因：宿主机开启 Secure Boot 导致内核模块签名校验失败
  • 处理：在 BIOS/UEFI 关闭 Secure Boot 后重启，再重新验证
Logs(gpu-operator/nvidia-driver-daemonset-d2cnd:nvidia-driver-ctr)[tail]
modprobe: ERROR: could not insert 'nvidia': Key was rejected by service

# 目前配置
root@prod-m1:/data/soft/gpu-operator# mokutil --sb-state
SecureBoot enabled

# 解决方法：
  # 重启进入BIOS，找到Secure Boot选项，选择Disable，保存并退出即可。
  # 在虚拟机启动出现 Proxmox 标志（或 TianoCore 标志）时，连续狂按 ESC 键进入 UEFI 设置界面。
  # 使用方向键找到 Device Manager -> Secure Boot Configuration。
  # 将 Attempt Secure Boot 这一项的勾选去掉（变成 [ ]）。
  # 按 F10 保存，按 ESC 退出并重启。

# 重启后验证
root@prod-m1:~# mokutil --sb-state
SecureBoot disabled
