非阻塞 IO（不推荐直接使用） #

  • server.setblocking(False) 设置非阻塞模式
  • 缺点1：需要不停轮询所有连接，浪费 CPU
  • 缺点2：连接越多，轮询时间越长，响应越慢
from socket import *

server = socket(AF_INET, SOCK_STREAM)
server.bind(('127.0.0.1', 8080))
server.listen(5)
server.setblocking(False)

conn_list = []
while True:
    try:
        conn, addr = server.accept()
    except BlockingIOError:
        pass
    else:
        conn.setblocking(False)
        conn_list.append(conn)
    del_l = []
    for conn in conn_list:
        try:
            data = conn.recv(1024)
        except BlockingIOError:
            pass
        except ConnectionResetError:
            del_l.append(conn)
        else:
            if not data:
                del_l.append(conn)
            else:
                conn.send(data.upper())
    for conn in del_l:
        conn_list.remove(conn)

select 模块 #

告诉内核监听哪些 IO 事件，内核帮筛选出就绪的事件，避免无效轮询。

  • rl readable list 可读事件列表
  • wl writable list 可写事件列表
  • xl exceptional list 异常事件列表
  • 超时参数：指定时间内无事件则返回空列表（非 sleep）
from socket import *
import select

server = socket(AF_INET, SOCK_STREAM)
server.bind(('127.0.0.1', 8080))
server.listen(5)
server.setblocking(False)

data_dic = {}
read_list = [server]
write_list = []

while True:
    rl, wl, xl = select.select(read_list, write_list, [])
    for sk in rl:
        if sk == server:
            conn, addr = sk.accept()
            read_list.append(conn)
        else:
            data = sk.recv(1024)
            write_list.append(sk)
            data_dic[sk] = data
    for sk in wl:
        sk.send(data_dic[sk].upper())
        data_dic.pop(sk)
        write_list.remove(sk)

epoll（Linux 专属） #

  • select 和 poll 都是轮询所有连接，效率 $O(n)$
  • epoll 采用事件驱动，类似回调函数，只关注有事件发生的连接，效率 $O(1)$
  • Linux 专属，Windows 不支持（Windows 只能用 select）
  • Python 高级库（asyncio、tornado、gevent）在 Linux 上会自动使用 epoll
